text

GitHub Repository
→ Settings
→ Secrets and variables
→ Actions
→ Variables
→ New repository variable
Add these.

Variable name	Example value	Required?	Purpose
DOCKERHUB_USERNAME	saibaba22	Yes	Your Docker Hub username
DOCKERHUB_REPOSITORY	sentinelops-lite	Yes	Docker Hub image repository
PYTHON_VERSION	3.11	Yes	Python version used in GitHub Actions
APP_HEALTH_PATH	/health	Yes	Application health endpoint
AWS_APP_NAME	sentinelops-lite	If using AWS	Elastic Beanstalk application name
AWS_ENV_NAME	sentinelops-lite-prod	If using AWS	Elastic Beanstalk environment name
AWS_REGION	ap-south-1	If using AWS	AWS deployment region
AZURE_WEBAPP_NAME	sentinelops-lite-prod	If using Azure	Azure App Service name
AZURE_RESOURCE_GROUP	sentinelops-rg	If using Azure	Azure Resource Group name
AI_PROVIDER	gemini	Recommended	Prometheus/Grafana provider label
AI_MODEL	gemini-2.5-flash	Recommended	Prometheus/Grafana model label
APP_VERSION	1.0.0	Recommended	Deployment metadata
ENVIRONMENT	production	Recommended	Deployment metadata/labels

B. GitHub Repository Secrets
Go to:

text

GitHub Repository
→ Settings
→ Secrets and variables
→ Actions
→ Secrets
→ New repository secret
Add these.

Secret name	Required?	Purpose
GEMINI_API_KEY	Yes	Gemini API key used by all three AI agents
DOCKERHUB_TOKEN	Yes	Docker Hub access token used to push images
AWS_ACCESS_KEY_ID	If using AWS	AWS deployment credential
AWS_SECRET_ACCESS_KEY	If using AWS	AWS deployment credential
AZURE_CREDENTIALS	If using Azure	Azure Service Principal JSON for azure/login@v2
MONITOR_API_URL_AWS	If using AWS	AWS deployed app’s /monitor/status URL
MONITOR_TOKEN_AWS	If using AWS	Secret token that protects AWS monitor endpoint
MONITOR_API_URL_AZURE	If using Azure	Azure deployed app’s /monitor/status URL
MONITOR_TOKEN_AZURE	If using Azure	Secret token that protects Azure monitor endpoint
